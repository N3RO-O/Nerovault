# Nerovault

Free, local-first budget & savings tracker. Single-page app, no backend, no accounts —
all data lives in the browser's `localStorage`.

## What changed from the CDN-based version

- **Tailwind and Chart.js are now self-hosted** (`assets/tailwind.css`, `assets/chart.umd.js`)
  instead of loaded from `cdn.tailwindcss.com` / `jsdelivr.net`. This removes a runtime
  dependency on third-party CDNs staying up, and Tailwind is now a pre-built ~18KB
  minified stylesheet instead of a JIT compiler running in every visitor's browser.
- **Fonts are self-hosted** (`assets/fonts/*.woff2`) instead of Google Fonts.
- **Offline support**: `manifest.json` + `sw.js` turn this into an installable PWA that
  caches its own app shell, so it keeps working with no network connection — which
  matters given the app's whole pitch is "your data stays on your device."
- **Focus trap** added for modals (Tab/Shift+Tab now stay inside an open modal instead
  of leaking focus onto the page behind it).
- **Versioned data migrations**: `loadData()` now runs saved data through a
  `MIGRATIONS` array keyed by `schemaVersion` instead of a pile of inline `if` patches.
  Imported backup files go through the same pipeline.
- **CSV export bug fix**: category names (free text) weren't being CSV-escaped, so a
  category containing a comma or quote could corrupt the exported file. Also, income
  rows now show the resolved category name instead of a raw internal ID.
- **Currency selector** now makes clear (tooltip + one-time toast) that switching
  currency relabels/reformats numbers — it does not convert amounts.
- Added `og:image`, Twitter Card tags, canonical link, and app icons for link previews.

## What changed in this update (v1.0.2)

Listening to early feedback, this release focuses on the Budget Categories experience,
a fresh app icon, and making the install-as-app option easy to spot.

- **Budget presets actually apply now.** Previously a preset (50/30/20, etc.) only landed
  on the four original default categories — if you'd renamed, removed, or added categories
  it silently did nothing. Now a preset redistributes across *your* current categories
  (fixed-₱ categories are left untouched), so it always works.
- **More presets + fully customizable.** 8 built-in presets (50/30/20, 70/20/10, 60/30/10,
  80/10/10, 40/30/20/10, 50/20/20/10, 60/10/10/20, 80/10/0/10). Tap **+ New** to save your
  own split, and edit/delete your custom presets with the icons on each chip.
- **Category colors now match the app theme.** The color picker uses Nerovault's own
  palette (sage/teal/lavender/amber/coral…) so picks never clash, and the popover is sized
  for mobile so it no longer stretches across the screen.
- **Set an exact peso amount, no percentages needed.** When editing a category you can
  switch it to "Fixed ₱" (e.g. ₱2,000) instead of a %. The form now shows a live
  ₱⇄% hint as you type, so you always see what a % is worth (and what % a peso is).
- **New app icon.** Responsive 192/512 + maskable + Apple-touch icons generated from the
  supplied `new icon.png`, so the app looks sharp on home screens and install prompts.
- **Install button stands out.** New visitors (not already running the installed app) see
  an emphasized, gently pulsing **Install app** button in the header — on desktop, Android,
  and iOS — so it's obvious you can install Nerovault. It stops showing once dismissed or
  installed.
- **Version shown in the footer** (currently v1.0.2).
- Added a **Budget presets** help section and refreshed the "Recently added" list.

## Rebuilding after a markup change

Tailwind's CSS is generated by scanning `index.html` for class names, so if you add or
change any Tailwind classes, rebuild the stylesheet:

```bash
npm install
npm run build:css
```

## Local dev server

```bash
npm run serve   # http://localhost:8080
```

Service workers work on `localhost` without HTTPS, so this is enough to test the
offline/install behavior locally. Opening `index.html` directly by double-clicking it
(`file://`) will **not** register the service worker — browsers require `http(s)`.

## Deploying so it's a real installable app

A service worker (and therefore "Add to Home Screen" / install) only works over
**HTTPS**, or on `localhost`. You need to host these files somewhere with HTTPS —
any static host works, since there's no backend. Free options, easiest first:

**Cloudflare Pages / Netlify (drag-and-drop, no account setup beyond signup)**
1. Run `npm install && npm run build:css` once, so `assets/tailwind.css` is fresh.
2. Go to the Pages/Netlify dashboard, choose "deploy manually", and drag the whole
   project folder in.
3. You get a `https://something.pages.dev` (or `.netlify.app`) URL with HTTPS already
   set up — done.

**GitHub Pages**
1. Push this folder to a GitHub repo.
2. Repo Settings → Pages → deploy from the `main` branch, root folder.
3. Your site is `https://yourusername.github.io/reponame/`. Because that's a
   **subpath**, not the domain root, and every asset link in this project is already
   relative (`./assets/...`, `./manifest.json`), it works as-is — no path edits needed.
4. Replace `REPLACE-WITH-YOUR-DOMAIN` in `index.html`'s `og:image`/`twitter:image`/
   `canonical` tags with your real GitHub Pages URL (those specific tags need a full
   absolute URL, unlike everything else in the file, since social platforms can't
   resolve relative image paths).

**Vercel** — `vercel deploy` from the folder also works with zero config, same idea.

## Installing it once it's hosted

- **Android / desktop Chrome, Edge, Brave**: an **Install app** button appears in
  Nerovault's own header once the browser decides the site is installable (valid
  manifest + service worker + HTTPS — the three things this project already has).
  You can also use the install icon in the browser's address bar.
- **iPhone/iPad (Safari)**: iOS has no install-prompt API, so there's no button to
  click — open the site in Safari, tap the Share icon, then **Add to Home Screen**.
  This is called out in the app's own Help → FAQ section too.
- Once installed, it opens in its own window with no browser chrome, has its own
  home-screen/app-drawer icon, and keeps working offline via the service worker.

## Updating a deployed/installed app later

Editing `index.html`, `assets/tailwind.css`, or anything else the service worker
caches won't reach people who already installed the app until you **bump
`CACHE_VERSION` in `sw.js`** (currently `nerovault-v2`) and redeploy. The old cache
gets cleared and the new files fetched automatically the next time they open the app
with a connection.

## Bumping the offline cache

`sw.js` caches the app shell by filename list. If you change any cached file's
*contents* (index.html, tailwind.css, chart.umd.js, fonts, manifest, icons), bump
`CACHE_VERSION` in `sw.js` so returning users' service workers pick up the update
instead of serving a stale cached copy.

## File map

```
index.html              the whole app (markup + inline JS)
manifest.json            PWA metadata (installable, icons, theme color)
sw.js                     service worker — offline app-shell caching
assets/
  tailwind.css            pre-built Tailwind output (generated, see build:css)
  fonts.css                @font-face rules for self-hosted Inter / JetBrains Mono
  fonts/*.woff2             font files
  chart.umd.js              Chart.js, vendored locally
  icon-192.png, icon-512.png, icon-192-maskable.png, icon-512-maskable.png, apple-touch-icon.png   app icons
  new icon/new icon.png    source artwork for the app icon
  og-image.png              social share preview image
tw-input.css              Tailwind entry file (@tailwind base/components/utilities)
tailwind.config.js       Tailwind content scanning config
```
