# Nerovault

Free, local-first budget & savings tracker. Single-page app, no backend, no accounts —
all data lives in the browser's `localStorage`.

## What changed from the CDN-based version

- **Tailwind and Chart.js are now self-hosted** (`assets/tailwind.css`, `assets/chart.umd.js`)
  instead of loaded from `cdn.tailwindcss.com` / `jsdelivr.net`. This removes a runtime
  dependency on third-party CDNs staying up, and Tailwind is now a pre-built ~18KB
  minified stylesheet instead of a JIT compiler running in every visitor's browser.
- **Fonts are self-hosted** (`assets/fonts/*.woff2`) instead of Google Fonts.
- **Offline support**: `manifest.json` + `sw.js` turn this into an installable PWA that
  caches its own app shell, so it keeps working with no network connection — which
  matters given the app's whole pitch is "your data stays on your device."
- **Focus trap** added for modals (Tab/Shift+Tab now stay inside an open modal instead
  of leaking focus onto the page behind it).
- **Versioned data migrations**: `loadData()` now runs saved data through a
  `MIGRATIONS` array keyed by `schemaVersion` instead of a pile of inline `if` patches.
  Imported backup files go through the same pipeline.
- **CSV export bug fix**: category names (free text) weren't being CSV-escaped, so a
  category containing a comma or quote could corrupt the exported file. Also, income
  rows now show the resolved category name instead of a raw internal ID.
- **Currency selector** now makes clear (tooltip + one-time toast) that switching
  currency relabels/reformats numbers — it does not convert amounts.
- Added `og:image`, Twitter Card tags, canonical link, and app icons for link previews.

## Rebuilding after a markup change

Tailwind's CSS is generated by scanning `index.html` for class names, so if you add or
change any Tailwind classes, rebuild the stylesheet:

```bash
npm install
npm run build:css
```

## Local dev server

```bash
npm run serve   # http://localhost:8080
```

## Bumping the offline cache

`sw.js` caches the app shell by filename list. If you change any cached file's
*contents* (index.html, tailwind.css, chart.umd.js, fonts, manifest, icons), bump
`CACHE_VERSION` in `sw.js` so returning users' service workers pick up the update
instead of serving a stale cached copy.

## File map

```
index.html              the whole app (markup + inline JS)
manifest.json            PWA metadata (installable, icons, theme color)
sw.js                     service worker — offline app-shell caching
assets/
  tailwind.css            pre-built Tailwind output (generated, see build:css)
  fonts.css                @font-face rules for self-hosted Inter / JetBrains Mono
  fonts/*.woff2             font files
  chart.umd.js              Chart.js, vendored locally
  icon-192.png, icon-512.png, apple-touch-icon.png   app icons
  og-image.png              social share preview image
tw-input.css              Tailwind entry file (@tailwind base/components/utilities)
tailwind.config.js       Tailwind content scanning config
```
